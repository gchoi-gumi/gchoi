## 🎯 방법 1: Vercel로 배포 (권장, 가장 쉬움)

### 1단계: GitHub에 코드 푸시
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/travel-app.git
git push -u origin main
```

### 2단계: Vercel 배포
1. https://vercel.com 접속
2. GitHub 계정으로 로그인
3. "Add New" → "Project" 클릭
4. GitHub 저장소 선택
5. **환경 변수 추가** (위의 3개 변수 복사해서 붙여넣기)
6. "Deploy" 클릭!

### 3단계: 완료!
- 약 2-3분 후 배포 완료
- URL: `https://your-app.vercel.app`

---

## 🌐 방법 2: Netlify로 배포

### 1단계: 로컬 빌드 (선택사항)
```bash
npm install
npm run build
```

### 2단계: Netlify Drop (가장 빠른 방법)
1. https://app.netlify.com/drop 접속
2. `build` 폴더를 드래그 앤 드롭
3. 완료!

### 또는 GitHub 연동
1. https://app.netlify.com 접속
2. "Add new site" → "Import an existing project"
3. GitHub 저장소 선택
4. **환경 변수 추가** (위의 3개 변수)
5. "Deploy site" 클릭!

## 🗺️ 배포 후 설정 (중요!)

### Kakao Map 도메인 등록
배포가 완료되면 **반드시** Kakao Developers에 도메인을 등록해야 지도가 표시됩니다:

1. https://developers.kakao.com 접속
2. 내 애플리케이션 선택
3. 플랫폼 → Web 플랫폼 추가
4. **배포된 URL 입력**:
   ```
   https://your-app.vercel.app
   또는
   https://your-app.netlify.app
   ```
5. 저장

> ⚠️ 도메인을 등록하지 않으면 지도는 표시되지 않지만, REST API로 모든 기능은 정상 작동합니다!

## 🐛 문제 해결

### 빈 화면이 나타남
**해결**: 환경 변수 확인
- Vercel/Netlify 대시보드에서 환경 변수 재확인
- `VITE_` 접두사 확인
- 재배포

### API 호출 실패
**해결**: 
1. 브라우저 콘솔(F12)에서 오류 확인
2. Supabase Edge Function이 배포되었는지 확인
3. 네트워크 탭에서 요청 URL 확인

### Kakao Map 오류
**해결**:
1. Kakao Developers에서 도메인 등록 확인
2. 정확한 URL 입력 (`https://` 포함)
3. 5-10분 대기 후 새로고침
